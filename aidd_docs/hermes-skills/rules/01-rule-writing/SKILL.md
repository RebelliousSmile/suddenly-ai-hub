# Rule: 01 Rule Writing

## Description

This rule defines rule: 01 rule writing. It applies to the project context.

## Context

## Content format

- Bullet points only, no prose
- 1 ultra short (3-7 words) rule per bullet
- Remove non-essential fluff
- Be concise, less is more

## Code references

- Existing code: use IDE include syntax (see ide-mapping rule)
- Globs: format without `@`
- Inline code: use backticks

## Groups

- Identify logical rule groups
- Not needed for short rules

## Examples (optional)

- Tiny generic example per group IF NEEDED
- Good code only

## Application

When working on this project, always follow these guidelines.
